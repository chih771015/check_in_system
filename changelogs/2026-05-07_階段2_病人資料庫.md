## 變更摘要

### 做了什麼

完整交付階段 2「病人資料庫」：後端 model / DTO / repository / service / handler / 路由，前端型別 / API client / admin 管理頁 / 歷史頁 placeholder / Menu 項目 + 路由註冊。

歷史紀錄聚合刻意留為空陣列，等階段 4 串入 SchedulePatient + DiagnosisPhoto 才實作真實資料；handler 與前端頁面結構已備妥，到時只需替換 service 內部。

### 後端

| 檔案 | 變動 |
|------|------|
| `backend/internal/model/patient.go` | 新增 Patient model，含 `(id_type, id_number)` unique index |
| `backend/internal/dto/patient.go` | 新增 CreatePatientRequest / UpdatePatientRequest / PatientResponse / TranslatorPatientResponse / PatientListQuery / PatientListResponse / PatientHistoryEntry / PatientHistoryResponse |
| `backend/internal/repository/patient_repo.go` | 新增 PatientRepository，含 `WithCtx(ctx)`、Create / Update / Delete / FindByID / FindByIDTypeAndNumber / List(search, page, pageSize) |
| `backend/internal/service/patient_service.go` | 新增 PatientService，CRUD + 重複檢查 (`ErrPatientDuplicate`) + `GetHistory` placeholder（TODO stage 4）+ `ListForTranslator`（TODO stage 3 收斂權限） |
| `backend/internal/handler/patient_handler.go` | 新增 admin CRUD + history endpoint + translator list endpoint，duplicate 回 409 |
| `backend/cmd/server/main.go` | AutoMigrate 加 Patient、初始化 PatientRepository / PatientService / PatientHandler、註冊 admin (`/admin/patients` CRUD + history) 與 translator (`/patients`) 路由 |

關鍵原則：
- IDNumber 入庫前 `strings.ToUpper`，重複檢查也比對大寫版本
- 全部 service 方法第一參數 `ctx context.Context`，內部 `repo.WithCtx(ctx)` 串 OTel trace
- duplicate 用 sentinel error，handler 統一 map
