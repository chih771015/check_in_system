## 變更摘要

### 做了什麼
回應 code review Finding 1 與使用者需求：

**1. 管理員可管理已完成病人的照片（修可達性缺口）**
- 先前 ScheduleManagement 排班詳情中，completed 病人只剩唯讀「查看照片」，使得注入的 admin 刪除/補傳能力（`adminListDiagnosisPhotos`/`adminDeleteDiagnosisPhoto`）在 UI 上無從觸及——而 completed 正是唯一有照片的狀態。
- 改為 completed 也顯示「管理照片」按鈕，開啟可增刪的 manage modal。管理員不受離開鎖定限制，是離開後的更正途徑。

**2. 翻譯員離開打卡後改為唯讀檢視**
- `DiagnosisUploadModal` 新增 `readOnly` prop：只顯示照片、隱藏新增輸入與逐張刪除鈕。
- `MySchedules` 以 `canEdit`（`checkinStatus === 'arrived'`）區分：離開後 completed 病人只剩「查看照片」（readOnly modal），不顯示上傳/刪除/未到入口。與後端 `DIAGNOSIS_LOCKED_AFTER_LEAVE` 一致，前端不再送出注定被擋的請求。

### 影響範圍
- 前端：`components/DiagnosisUploadModal`、`pages/translator/MySchedules`、`pages/admin/ScheduleManagement`。
- 文件：COMPONENTS_SPEC、PAGES_SPEC、PRODUCT_SPEC、TEST-PLAN。

### 測試
- 新增 `DiagnosisUploadModal` readOnly 單元測試（顯示照片、無檔案輸入、無刪除/送出鈕）。
- 前端 `vitest` 80 全過、`tsc`/`eslint` 乾淨。

### 注意事項
- code review Finding 2（`assertNotLeftYet` 對非 NotFound DB 錯誤 fail-open）屬低 severity，本次未改；如要 fail-closed 可改用 `errors.Is(err, gorm.ErrRecordNotFound)` 區分。
- 翻譯員離開後唯讀為前端 UX；後端仍以 409 為最終守衛。
