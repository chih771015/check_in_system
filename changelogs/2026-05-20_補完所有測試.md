## 效能優化（重要）

**問題**：`userEvent.type` 在 antd Form 內每個 keystroke 都觸發 React re-render
+ form validation，多測試並行時容易超過 15s timeout。

**解法**：填表場景改用 `fireEvent.change(input, { target: { value } })`
一次塞值，速度差到一個數量級。`user.click` 維持不變（觸發事件處理流程）。

**測試間 portal 污染**：antd 的 `Modal.confirm` 與 `message` 透過 portal 渲染
到 `document.body`，跨 test 不會被 RTL 自動清掉。在每個 RTL test 檔加：
```ts
beforeEach(() => { document.body.innerHTML = ''; ... });
afterEach(() => { cleanup(); document.body.innerHTML = ''; });
```
