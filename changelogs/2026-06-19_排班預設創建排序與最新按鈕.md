## 做了什麼

### 後端
- `ScheduleRepository.FindRecentByCreated(limit)`：`created_at DESC, id DESC`（id 作穩定 tiebreaker）取前 N 筆，preload Translator + Patients.Patient。
- `ScheduleService.List`：當**無任何篩選**（translatorID=0 且 dateFrom/dateTo/location 皆空）→ 走 `FindRecentByCreated(100)`（預設模式）；**有任一篩選** → 沿用 `FindAll`（date ASC，不限筆數）。常數 `DefaultRecentScheduleLimit = 100`。

### 前端（`ScheduleManagement.tsx`）
- 新增「最新創建排班」按鈕：`isDefaultMode`（filters 為空）時 `type="primary"`（高亮），否則一般樣式。
- 套用任一篩選（日期區間 / 翻譯員 / 地點）→ 離開預設模式、按鈕取消高亮、依篩選查詢。
- 清除所有篩選 → 回到預設模式且高亮。
- 按下按鈕 → `resetToDefault()`：清空 filters + 以 `filterKey` 強制 remount 三個篩選輸入（RangePicker / 翻譯員 Select / 地點 Search）使其顯示值一併清空，並回到高亮。
- i18n：`schedules.latestCreated`（zh-TW/en/th）。

### 順手修正（與本需求直接相關）
- 排班日期篩選的 query key 由 `startDate/endDate` 改為後端實際綁定的 `dateFrom/dateTo`（`ScheduleListQuery` form tag）。原本不一致導致日期篩選送到後端不會生效、也無法正確離開預設模式；與 `CheckinRecords` 既有用法一致。
