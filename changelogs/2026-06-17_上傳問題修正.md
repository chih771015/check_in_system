## 變更摘要

實測回報四個問題，逐一查證：

### 問題 1 & 2：送出後 network error / 太大、且送出鈕沒變灰 → 環境根因
- 根因：對外穿透（expose overlay）掛載的是 `nginx.expose.conf`，先前只改了 `nginx.conf`，所以實際擋上傳的 nginx 仍是 **20m**。上傳超過 20MB → 413/network error；前端門檻是 90MB，沒到 90MB 自然不變灰、可按 → 送出後才被 nginx 擋。
- 修正：`nginx.expose.conf` 同步改 100m。**需重啟 expose stack 才生效**。
- 前端送出鈕的 oversize 邏輯本身正確：nginx 100m 後，>90MB 會正確變灰。

### 問題 3：選檔後無法移除個別選取 → 已修
- `DiagnosisUploadModal` 選取清單改成「每張一列 + 移除鈕」，送出前可剔除選錯的。
- 新增 i18n `diagnosis.removeSelected`（三語系）+ 單元測試。

### 問題 4：Alice 的 preview 沒照片 → 非 bug（資料/檔案/serving 皆正常）
- 查證：DB 有 10 筆 diagnosis_photos；檔案存在於 backend 容器 `/app/uploads`（Jun 13 那批為真實 0.6–1MB jpeg）；直連 backend:8080 與經 frontend:3000 取圖皆 HTTP 200。
- completed 的 schedule_patient（sp 3/5/7/13/15）都有照片；看到沒照片者應為 pending/no_show 格子（本就無照片）或舊 build。依指示「單純沒照片就不管」，不處理。

## 影響範圍
- 基礎設施：`docker/nginx.expose.conf`。
- 前端：`components/DiagnosisUploadModal`、`i18n`(3 語系)。

## 注意事項（部署/重啟）
- **要讓 nginx 100m 生效**：重啟 expose stack，例如
  `docker compose -f docker/docker-compose.yml -f docker/docker-compose.expose.yml up -d --build frontend`
  或重跑 `scripts/expose.sh`。
- 同時 `--build` 才會把前端新版（移除鈕）打進 image。
- 一般單機部署（非 expose）用的是 `nginx.conf`，先前已改 100m。
