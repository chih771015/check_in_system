## 變更摘要

### 做了什麼

針對「翻譯員打卡紀錄」與「排班管理」回報的三個問題：

1. **補打卡不再被誤判遲到／準時**（後端）
   `MyStats` 計算到達準時度時，會拿 `CheckinTime` 跟排班開始時間比。但補打卡的
   `CheckinTime` 是補登當下時間（`time.Now()`），不是實際到場時間，比對毫無意義，
   導致一筆補打卡會同時被算進「補打卡」又被假性算進「遲到」。改為遇到補打卡直接跳過
   準時/遲到判定，使三個維度乾淨：到達分準時/遲到（僅當場即時打卡），補打卡自成一類。

2. **排班編輯帶回時間**（前端 bug）
   `openEdit` 先前只回填 translator/location/note，漏掉 date/startTime/endTime，
   導致編輯視窗一開日期與起訖時間欄位全空。補上三欄並將 API 字串轉為 dayjs。

3. **打卡統計卡重新分組**（前端 UX）
   原本 6 張卡平鋪一排易被誤解成可加總。改為依三維度分組（打卡總計 / 準時度 / 補打卡），
   各組加小標題與說明，並補齊三語 i18n。

### 影響範圍

- 後端：`internal/service/checkin_service.go`（MyStats）、對應測試
- 前端：`pages/admin/ScheduleManagement.tsx`（編輯回填）、`pages/translator/MyCheckins.tsx`（統計卡）
- i18n：`zh-TW / en / th` 三語 locale（新增 5 個 stats 字串）

### 注意事項

- **行為變更**：補打卡的到達自此不再計入「準時」或「遲到」。歷史數據若曾把補打卡
  當作遲到/準時，重新查詢時數字會變化（這是修正，非破壞）。
- 「準時」門檻維持為排班開始時間 + 5 分鐘；「遲到/準時」僅針對「到達」卡，不含離開卡。
- 驗證：後端 `go test ./...` 全過；前端 `npm run typecheck` 通過、`npm test` 通過
  （唯一一次 `DiagnosisResults.test.tsx` 在滿載下超時，單獨重跑即過，與本次改動無關）。
