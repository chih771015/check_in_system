# 更動報告 — 2026-05-07：補後端單元測試

## 背景

階段 1 + 階段 2 完工後，使用者點出**完全沒有測試**（前後端 0 個 test 檔），驗證僅靠「go build 通過 + tsc 通過」。本次補上後端核心邏輯的單元測試。

## 完成項目

### 新增測試依賴
- `github.com/stretchr/testify`：assertion / require helpers
- `gorm.io/driver/sqlite`：in-memory sqlite 作為 DB fake，每個測試一個隔離實例

### 測試檔案

**`internal/service/testutil_test.go`**（共用）
- `newTestDB(t)`：建立 in-memory sqlite + AutoMigrate 所有 model

**`internal/service/patient_service_test.go`**（9 個 case）
- `Create_NormalizesIDNumber`：IDNumber 入庫前 trim + uppercase
- `Create_DuplicateRejected`：同 IDType+IDNumber 拒絕（含 case-insensitive）
- `Create_DifferentIDTypeAllowed`：同號碼但不同 IDType 允許
- `Update_SelfExclude`：編輯自己保留同 IDNumber 可成功
- `Update_CollidesWithOther`：撞到別人的號碼會拒絕
- `Update_NotFound` / `Delete_NotFound`：missing record 回 sentinel
- `GetHistory_EmptyButValid`：階段 2 contract（空陣列）
- `NormalizeIDNumber`：純函式表格測試

**`internal/service/admin_service_test.go`**（6 個 case）
- `CreateAdmin_DuplicateEmail`：重複 email 拒絕
- `CreateAdmin_Success_ForcesPasswordChange`：新建 admin must_change_pw=true
- `DeleteAdmin_CannotDeleteSelf`：自刪保護
- `DeleteAdmin_TargetNotAdmin`：非 admin 不可被此 endpoint 刪
- `DeleteAdmin_Success`：成功刪除
- `ListAdmins`：只回 role=admin 的使用者

**`internal/handler/error_mapper_test.go`**（3 個 test，含 24 個 subtests）
- `MapError_KnownSentinels`：24 個 sentinel error → (status, code) 對應
- `MapError_UnknownDefaultsToInternal`：未知 error → 500 + INTERNAL_ERROR
- `MapError_WrappedSentinel`：`%w` 包裝後 `errors.Is` 仍能識別

**`internal/dto/error_test.go`**（2 個 case）
- `NewError_FieldsAndJSON`：結構欄位 + JSON 序列化形狀
- `ErrorCodes_Stability`：關鍵錯誤碼字串值固定（避免改動破壞前端 i18n key）
