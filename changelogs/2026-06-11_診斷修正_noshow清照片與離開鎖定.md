## 變更摘要

### 做了什麼
回應 code review 的兩項發現，並依使用者決策修正：

**1. 標記 no_show 時清空既有照片（按錯更正）**
- 問題：completed（有照片）→ no_show 後照片仍殘留；刪掉殘留的最後一張會把狀態洗回 pending、連帶清掉 no_show 原因。
- 修法（使用者決策「直接把照片刪了就好」）：`MarkNoShow` / `AdminMarkNoShow` 在轉 no_show 前先 `purgePhotos`（刪 DB 列 + best-effort 刪檔）。no_show slot 因此永不殘留照片，刪除退回 pending 只會發生在 completed slot。

**2. 離開打卡後鎖定翻譯員的診斷修改（僅管理員可改）**
- 問題：已 leave 仍可經 API 改診斷，造成「已離開但有 pending 病人」的不一致。
- 修法：`DiagnosisService.WithCheckinRepo` 注入 checkinRepo；`assertNotLeftYet` 在排班已有 `leave` 打卡時，讓 translator 的 upload/delete/no_show 回 `DIAGNOSIS_LOCKED_AFTER_LEAVE`（409）。唯讀列表與所有 `Admin*` 不受限。前端離開後本就隱藏按鈕，此為後端 defense-in-depth。

順帶補上前次遺漏的 `DIAGNOSIS_PHOTO_NOT_FOUND` 三語系 i18n。
