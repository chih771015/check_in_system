## 變更摘要

### 完整補打卡邏輯說明（修復後）

```
排班日當天，排班結束前
  → checkinStatus=none    → [到達打卡] 正常流程
  → checkinStatus=arrived → [離開打卡] 正常流程

排班日當天，排班結束後 / 排班日已過
  → checkinStatus=none    → [補打卡(到達)] 補打卡流程（需填原因）
  → checkinStatus=arrived → [補打卡(離開)] 補打卡流程（需填原因）
  → checkinStatus=makeup  → [補打卡(離開)] 到達補打卡完成，繼續補離開

到達 + 離開 均已完成（不論正常或補打卡）
  → checkinStatus=completed → [已完成] 無按鈕
```

### 修復的三個 Bug

#### Bug 1 — `isPast()` 只比較日期（`frontend/src/pages/translator/MySchedules.tsx`）

**問題**：`schedule.date < today` 以「天」為單位，同一天的排班即使已過結束時間仍顯示正常打卡按鈕。例如排班 09:00–12:00，下午 15:00 進來仍看到「到達打卡」而非「補打卡(到達)」。

**修復**：同一天時追加比對 `endTime`，當下時間超過 `HH:MM` 才切換為補打卡按鈕。

#### Bug 2 — `makeup` 狀態無對應按鈕（`frontend/src/pages/translator/MySchedules.tsx`）

**問題**：`renderActions()` 沒有處理 `checkinStatus === 'makeup'` 的情況，回傳 `null`。翻譯員補打卡到達後，無法看到「補打卡(離開)」按鈕，流程卡死。

**修復**：補上 `makeup` 狀態的處理，顯示「補打卡(離開)」按鈕。

#### Bug 3 — `getCheckinStatus` 順序錯誤（`backend/internal/service/schedule_service.go`）

**問題**：`hasMakeup` 判斷在 `hasArrive && hasLeave` 之前，當到達和離開都以補打卡完成時，回傳 `"makeup"` 而非 `"completed"`，導致「已完成」標籤永遠不會出現。

**修復**：調整判斷順序，先判斷 `hasArrive && hasLeave → completed`，再判斷 `hasMakeup → makeup`。
