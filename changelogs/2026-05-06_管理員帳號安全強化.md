## 變更摘要

### 背景

針對三個安全與營運缺口進行修補：
1. JWT_SECRET 使用預設弱密碼可被偽造 token
2. Admin 忘記密碼完全沒有恢復路徑
3. 只有一個 admin 帳號，且前端沒有修改密碼入口

---

### 功能一：JWT_SECRET 強制驗證

**問題**：`JWT_SECRET=dev-secret-key-change-in-production` 預設值讓任何人可偽造 admin token。

**修改**：
- `backend/internal/config/config.go`：`config.Load()` 啟動時檢查 JWT_SECRET，若為預設值或長度 < 32 字元則印 FATAL 並 `os.Exit(1)`
- `docker/docker-compose.yml`：JWT_SECRET 改為明確必填（`${JWT_SECRET}`），不提供 fallback
- `backend/.env`：替換為 `openssl rand -hex 32` 產生的強密碼（64 字元 hex）

---

### 功能二：CLI 緊急重設密碼

**問題**：Admin 忘記密碼沒有任何恢復路徑，只能進 DB。

**修改**：
- `backend/cmd/server/main.go`：新增 `-reset-password` flag
  - 使用方式：`docker exec thai-backend ./server -reset-password <email> <newPassword>`
  - 密碼長度 < 8 字元拒絕執行
  - 重設後自動設 `must_change_pw=true`

---

### 功能三：Admin 修改密碼 UI

**問題**：後端 `POST /api/auth/change-password` 已存在，但前端 admin 介面沒有入口。

**修改**：
- `frontend/src/components/AppLayout.tsx`：Header 加「修改密碼」按鈕（僅 admin 可見）
  - Modal：舊密碼 + 新密碼（≥8字元）+ 確認密碼
  - 成功後更新 token，不需重新登入

---

### 功能四：多管理員帳號管理

**問題**：只有 `seedAdmin()` 能建立 admin，無法透過 UI 管理多個管理員。

**新增後端 API**：
- `GET  /api/admin/admins` — 列出所有管理員
- `POST /api/admin/admins` — 建立新管理員（`email`, `name`, `password`）
- `DELETE /api/admin/admins/:id` — 刪除管理員（禁止自刪，前後端雙層防護）

新帳號自動設 `must_change_pw=true`，首次登入後強制改密碼。

**新增前端頁面** `/admin/admins`（側邊欄「帳號管理」）：
- 管理員列表 + 建立 Modal + 刪除確認
- 自己的帳號刪除按鈕 disabled，顯示 Tooltip 說明
