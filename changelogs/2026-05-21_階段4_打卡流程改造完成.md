## 已知限制與後續

1. **Admin 代補登 UI 未整合**：後端 endpoints 已存在（`POST /api/admin/diagnosis` / `no-show`），但 admin `CheckinRecords.tsx` 還沒加按鈕直接呼叫。可在排班管理頁的詳情 modal 補。
2. **舊環境照仍可顯示**：admin `CheckinRecords.tsx` 詳情 modal 中 `environmentUrl` 非空時仍顯示縮圖（向後相容）。
3. **診斷證明保留期限永久**：未進入 `cleanup_service` 的刪除範圍，與需求一致。
4. **照片大小 / MIME 後端未驗證**：僅前端 `accept="image/*"`；惡意上傳 PDF 仍能存進去。後續加 MIME 白名單。
