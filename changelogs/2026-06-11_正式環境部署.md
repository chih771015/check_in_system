## 變更摘要

### 做了什麼
為「部署到一台正式 server」備齊所需的設定檔、容器編排與一鍵部署腳本。依使用者情境拍板的決策：

| 項目 | 選擇 |
|------|------|
| Server | 診所內部機器（無公開 IP） |
| 資料庫 | compose 內建 PostgreSQL 容器 |
| 對外方式 | Cloudflare Tunnel（自動 HTTPS、免開公開 port） |
| 選填整合 | SMTP、Google Sheets、LINE、Jaeger 全部啟用 |
| CORS | 維持現狀（前後端同源，風險低，不動 production code） |

新增檔案：
- `backend/.env.production.example` — 正式環境變數範本，全中文註解。
- `docker/docker-compose.prod.yml` — 正式環境專用編排。
- `scripts/deploy-prod.sh` — 一鍵部署腳本（自動產生密鑰 → 驗證 → 建置 → 等待就緒 → 印登入資訊；支援 `--tunnel`）。
- `docs/PRODUCTION_DEPLOY.md` — 白話部署步驟書。
- `.gitignore` — 補上正式環境密碼/憑證的忽略規則。

### 影響範圍
- 部署 / 基礎設施：`docker/`、`scripts/`、`backend/.env.production.example`、`.gitignore`。
- 文件：`docs/PRODUCTION_DEPLOY.md`、本 changelog。
- **未更動任何 Go / 前端原始碼**，因此無需新增測試。

### 注意事項
- **安全**：`backend/.env.production`（實際密碼檔）已加入 `.gitignore`，切勿提交；範本 `.example` 仍正常追蹤。
- **對外 port 收斂**：正式 compose 中 postgres、backend、Jaeger OTLP 收集埠皆不對外；前端與 Jaeger UI 僅綁 `127.0.0.1`，對外公開交給 Cloudflare Tunnel。
- **待使用者確認**：
  1. 網域目前由誰管理（決定 Cloudflare Tunnel 怎麼設）。
  2. 選填整合金鑰（SMTP / Google JSON / LINE token）尚未填；未填則該功能不運作，不影響系統本體。
- 升級流程：拉新 code 後重跑 `bash scripts/deploy-prod.sh` 即可，資料（postgres volume、uploads）保留。
- 驗證紀錄：`docker compose -f docker/docker-compose.prod.yml config` 通過（含 tunnel profile）；對外 port 綁定與 `.gitignore` 行為均已實測確認。
