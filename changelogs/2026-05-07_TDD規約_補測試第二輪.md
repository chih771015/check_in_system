### 2. 後端 AuthService 測試（13 cases）
**Login**：
- 成功 → 回 token + 正確 user
- 錯密碼 → ErrInvalidCredentials + login_attempts++
- 未知 email → 同 ErrInvalidCredentials（不洩漏帳號存在）
- 停用帳號 → ErrAccountDisabled
- 鎖定中 → ErrAccountLocked
- 第 5 次失敗 → 自動鎖定 LockedUntil
- 成功登入 → 重置 attempts + 清 LockedUntil

**ChangePassword**：
- 成功 → 回新 token + must_change_pw=false + bcrypt 驗證新 hash
- 舊密碼錯 → ErrOldPasswordIncorrect
- User not found → ErrUserNotFound

**AdminResetPassword**：
- 自己重設自己 → ErrCannotResetSelf
- target not found → ErrUserNotFound
- 成功 → target must_change_pw=true + 新 hash 可驗證

### 3. 前端測試框架（vitest + happy-dom）
**依賴**：
- vitest 4 / @testing-library/react / @testing-library/jest-dom / @testing-library/user-event
- happy-dom（取代 jsdom — node 25 + vitest 4 對 jsdom 不友善）

**設定**：

- `src/test/setup.ts`：載入 jest-dom matchers、polyfill `localStorage` / `sessionStorage`（node 25 內建版在 vitest 4 環境失效）+ `matchMedia` polyfill
- `package.json` scripts：`npm test`、`npm test:watch`

### 4. 前端測試（16 cases）

**`src/i18n/__tests__/i18n.test.ts`（6）**：
- 支援語言列表 = `['en', 'zh-TW', 'th']`
- 預設語言 fallbackLng 含 `en`
- 三語各自正確翻譯 `errors.INVALID_CREDENTIALS`
- `setLanguage()` 寫入 localStorage 並切換
- 未知 code 用 `defaultValue` fallback
- 三語 key 集合完全對齊（無漏譯）

**`src/api/__tests__/client.test.ts`（10）**：
- `unwrapResponse` 三種情境（單鍵剝殼 / 多鍵保留 / primitive 不動）
- `mapErrorResponse` 翻譯英文 / 切換到中文後翻譯成中文
- 未知 code 用 backend message fallback
- 401 → 清 token+user + redirect /login
- 403 PASSWORD_CHANGE_REQUIRED → redirect /change-password
- 已在 /change-password 時不再 redirect
- 空 response 不會 crash

### 5. 重構（為了可測）
**`api/client.ts`**：
- 把錯誤處理抽成 `mapErrorResponse()` 與 `unwrapResponse()` 兩個公開函式
- `mapErrorResponse` 接受 `redirect` 與 `currentPath` 兩個 dependency injection 參數，預設用 window.location
- interceptor 包裝這兩個函式，行為不變
