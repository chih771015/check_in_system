## 變更摘要

### 做了什麼

1. **金額移除 NT$ 前綴**：依需求先拿掉金額前的幣別符號，只顯示千分位數字，幣別之後再定。
   - `formatNT` 共用函式移除 `"NT$ "` 前綴。
   - `PatientHistory` 的 `Statistic` 移除 `prefix="NT$"`。
2. **操作紀錄 ID 不一致修正**：建立類操作（create_translator / create_admin / create_schedule）原本把 audit log 的 `target_id` 寫死成 `0`，與 `create_patient` 不一致，導致後台操作紀錄一堆 ID 顯示 0、無法追溯。改為記錄新建立物件的真實 ID。

### 影響範圍

- 後端 service：`TranslatorService.Create`、`AdminService.CreateAdmin`（簽章改為 `(uint, error)`）
- 後端 handler：`translator_handler`、`admin_handler`、`schedule_handler` 的建立流程
- 後端測試：`translator_service_test`、`admin_service_test`
- 前端：`utils/money.ts`、`utils/money.test.ts`、`pages/admin/PatientHistory.tsx`

### 注意事項

- **破壞性變更（內部）**：`TranslatorService.Create` 與 `AdminService.CreateAdmin` 簽章由 `error` 改為 `(uint, error)`，所有呼叫端與測試已同步更新。
- 匯入類操作（`import_schedules` / `import_patients`）為批次處理，無單一目標 ID，`target_id` 維持 `0`（屬合理設計）。
- `formatNT` 函式名暫予保留，方便日後決定幣別時定位；若要正名為 `formatMoney` 需一併調整 3 個呼叫端。
- 驗證：後端 `go build ./...` + `go test ./...` 全綠；前端 `npm run typecheck` 通過、`money.test.ts` 通過。
