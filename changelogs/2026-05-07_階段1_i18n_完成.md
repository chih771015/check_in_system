## 變更摘要

### 完成項目

**1. 後端錯誤碼化（已先前 commit）**
- `dto/error.go`：`ErrorResponse{Code, Message}` + 50+ 錯誤碼常量 + `NewError()` helper
- `service/*.go`：sentinel error（admin / auth / checkin / schedule / translator / patient）
- `handler/error_mapper.go`：`respondError` / `respondBadRequest` / `respondCode` helper + `mapError` 集中映射
- 所有 handler 改用新 helper（auth / admin / translator / schedule / checkin / patient / audit / export_schedule）

**2. 前端基礎建設**
- 安裝 `react-i18next` + `i18next`（不偵測瀏覽器語言，預設 `en`，localStorage 持久化）
- `src/i18n/index.ts`：初始化 + `setLanguage()` helper + `SUPPORTED_LANGUAGES` 常量
- `src/i18n/locales/en.json` / `zh-TW.json` / `th.json`：完整三語翻譯，~140 keys
- `main.tsx`：import i18n 初始化
- `App.tsx`：antd `ConfigProvider` 動態切換 locale (`enUS` / `zhTW` / `thTH`)
- `components/AppLayout.tsx`：Header 右上加 `<Select>` 語言切換器
- `api/client.ts`：axios interceptor 攔 `{code, message}`，用 `i18n.t(\`errors.${code}\`)` 翻譯
- `types/index.ts`：新增 `ApiError` 型別

**3. 前端頁面 i18n（12 頁全部完成）**
- 通用：Login、ChangePassword
- Admin：AdminManagement、PatientManagement、PatientHistory、TranslatorManagement、ScheduleManagement、CheckinRecords、ExportSettings、AuditLogs
- Translator：MySchedules、MyCheckins、CheckIn、MakeupCheckIn
- GeoStatusBlock 內的 iOS/Android/Chrome 權限提示全部多語化
- Excel 匯入範本標題改英文

### 注意事項
- 部分 i18n keys 為了重用而共用語意接近的 key（例如「補打卡」與「補打卡原因」同用 `checkins.makeupReason`）
- 舊 `cleanupService` import 在 main.go 仍存在，不影響本次變更
- 排班/打卡頁面內仍有少數英文 placeholder（重複規則選項、日期區間 placeholder），這些非關鍵 UI 字串可後續再補
