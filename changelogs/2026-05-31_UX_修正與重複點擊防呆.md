## 變更摘要

### 新功能：診斷結果總覽
- **後端**：`GET /api/admin/diagnosis-results` JOIN schedule_patients + schedules + users + patients
  - 只回 `completed` + `no_show`，排除 pending
  - 排序 `schedule.date DESC, sp.start_time DESC`（依使用者 Q2 決定）
  - filter：dateFrom / dateTo / translatorId / status / patientName
  - 照片批次撈避免 N+1
  - TDD 9 cases
- **新 endpoint**：`GET /api/admin/schedule-patients/:id/photos` 給排班詳情 modal 點看照片用（TDD 3 cases）
- **前端頁面**：`/admin/diagnosis-results`
  - 預設顯示最近 7 天
  - 病人欄含 📞 電話 / ID 類型+號碼
  - 照片以「N 張」按鈕觸發 Modal + `Image.PreviewGroup` 放大檢視
  - 排在 admin 選單「打卡紀錄」與「定期匯出設定」之間

### UX 改善
1. **手機 sidebar 點選自動收起**（`Grid.useBreakpoint`，< lg 才收）
2. **MyCheckins 統計卡手機跑版**：`Col span={4}` 寫死 → 響應式 `xs/sm/md/lg` + 垂直 gutter + Card size="small" + Table x-scroll
3. **`View History` vs `Visit History` 不一致**：統一 `patients.history`，三語 key 同步移除 `viewHistory`
4. **病人歷史照片放大**：plain `<img>` → `Image.PreviewGroup`
5. **`No visit records yet... future release` 過時文案**：階段 4 已實作，改為中性「目前無看診紀錄」
6. **排班詳情 modal「操作」欄混雜**：拆成
   - 「診斷證明」欄：completed→查看照片按鈕 / no_show→紅字原因 / pending→—
   - 「操作」欄：只放 action 按鈕
7. **病人卡顯示完整資訊**：CheckIn、MySchedules、ScheduleManagement 詳情 modal 病人欄加 📞 電話 + ID 類型+號碼

### Bug 修
1. **413 File Too Large**：nginx `/api/` location 沒設 `client_max_body_size`，吃預設 1MB；手機照片 3-8MB → 改 20MB
2. **病人歷史照片不顯示（網址 404）**：nginx 沒 proxy `/uploads/`，落到 SPA fallback；補 proxy 規則
3. **登入失敗清空表單 + 沒訊息**：axios interceptor 401 觸發 `window.location.href = '/login'` 整頁 reload，吞掉 message toast；加 guard：請求是 `/auth/login` 或當前已在 `/login` → 不 redirect
4. **錯誤訊息看不到原因**：抽 `extractApiError` 共用 helper，套用各處 catch；之後失敗會顯示後端翻譯後的具體訊息而非 generic "Failed"
5. **刪除排班 FK violation**：cascade 沒清 schedule_patients；補上，sqlite 測試 DB 也開啟 `PRAGMA foreign_keys`
6. **CheckIn 沒顯示病人清單**：只顯示 `schedule.patientName` 舊欄位，改為 iterate `schedule.patients[]`
7. **自拍只能拍照不能從相簿選**：移除 `capture="user"` 屬性
8. **no_show 後無法改回**：MySchedules `showActions` 條件從 `status === 'pending'` 改為 `status !== 'completed'`

### Add Schedule modal 五項 UX 修正
- A. 「Failed」訊息看不到原因 → `extractApiError`
- B. overall 時段變動後病人列不縮回範圍 → `useEffect` + `clampPatientTimes`
- C. 病人時段沒前端防呆 → `validatePatientTimes` 鏡像後端規則
- D. ListEditor 拿不到新 overall（隱藏 bug） → `Form.useWatch`
- E. `recurrenceRule` / `recurrenceUntil` label key 用錯（顯示 Show History / End Time）→ 補三語 key + 改 label + 下拉選項 i18n 化

抽 `utils/schedulePatient.ts` 含 clamp / validate 純函式（13 cases）+ `utils/apiError.ts`（5 cases）。
