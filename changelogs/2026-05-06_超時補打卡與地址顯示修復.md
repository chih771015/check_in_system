## 變更摘要

### 問題 1：超過排班時間打卡沒有被標記為補打卡

**根因**：`checkin_handler.go` 呼叫 `checkinService.Checkin(...)` 時固定傳入 `isMakeup=false`。Service 層收到後直接建立紀錄，沒有比對打卡時間與排班結束時間。

**修復（`checkin_service.go`）**：在建立打卡紀錄前，加入時間比對：
- 解析排班的 `Date + EndTime` 成 `time.Time`
- 若 `time.Now().After(endLocal)` 且 `isMakeup=false` → 自動設 `isMakeup=true`
- `makeupReason` 填入「打卡時間超過排班結束時間（系統自動標記）」作為系統備註
- 管理員在稽核頁可見到自動標記的補打卡原因

### 問題 2：打卡頁定位只顯示經緯度數字

**根因**：前端 `useGeolocation.ts` 取得座標後，`address` 被設為 `"25.033000, 121.565400"` 格式的字串。後端的 Nominatim reverse geocoding 條件是 `if address == ""`，由於前端傳入非空字串，反向地理編碼從未執行。

**修復**：
- `useGeolocation.ts`：新增 `reverseGeocode(lat, lng)` 函式，直接在前端呼叫 Nominatim
- 取得 GPS 座標後，先顯示座標（立即回饋），Nominatim 回傳後換成文字地址（如「台北市信義區松壽路…」）
- 前端 `address` state 同步更新，打卡送出時一併傳給後端儲存
- 後端的 Nominatim 呼叫保留作為最終 fallback（若前端傳來空字串時）

### 影響範圍

| 範圍 | 檔案 |
|------|------|
| 後端 Service | `backend/internal/service/checkin_service.go` |
| 前端 Hook | `frontend/src/hooks/useGeolocation.ts` |

### 注意事項

- 「系統自動標記」的補打卡 `makeupReason` 可由管理員在打卡紀錄管理頁手動修改
- Nominatim 為免費服務，有使用頻率限制，失敗時前端 fallback 顯示座標，打卡不受影響
- 超時判斷以**伺服器時間**為準（`time.Now()`），非前端時間，不可被前端竄改
