# 需求規格 — 金額統計與排班預設排序

- 日期：2026-06-19
- 分支：`feature/local-expose`
- 狀態：Spec（待 plan 後實作，本次只開規格文件）

## 0. 名詞與金額定義（全域）

- **金額一律指 `ActualAmount`（實付）**：翻譯員出訪後填入的實際支付金額（整數 TWD）。
  - `no_show` 狀態的 `SchedulePatient` 其 `ActualAmount = 0`，故自然不計入任何加總。
  - `PrepaidAmount`（預付）不在本批需求的任何統計／顯示中使用。
- **資料來源**：`schedule_patients.actual_amount`，必要時 JOIN `schedules`（取 `date`）與 `patients`（取病人）。
- **「月份／年份」一律以 `schedules.date`（看診日期）判定**，不是 `created_at`。

## 1. 後台全域橫幅：當月病人總支出

### 背景
目前 admin 登入後直接導向 `/admin/translators`，沒有首頁/dashboard。

### 需求（已與需求方確認）
- **不另開 dashboard 頁**。改為在**所有後台（admin）頁面最上方**都看得到的一條橫幅。
- 橫幅顯示**「當下日曆月份」的所有病人實付總額**（單一數字，非每月明細）。
  - 例：今天 2026-06-19 → 顯示 2026 年 6 月（`date` 落在 2026-06-01 ~ 2026-06-30）所有 `schedule_patients.actual_amount` 加總。
- 僅 admin 角色看得到；翻譯員頁面不顯示。

### 行為細節
- 進入任一後台頁面時載入；金額為當下查詢時的即時值。
- 顯示格式範例：`本月病人總支出（2026-06）：NT$ 123,456`。
- 查無資料時顯示 `NT$ 0`。
- 載入中顯示 loading 占位；查詢失敗顯示可重試或淡化提示，不可阻擋頁面其他功能。

### 驗收條件
- [ ] admin 在任一後台頁面頂部都能看到當月實付總額。
- [ ] 總額 = 當前日曆月份內（依 `schedules.date`）所有 `schedule_patients.actual_amount` 之和。
- [ ] `no_show`（actual=0）不影響結果。
- [ ] 翻譯員角色看不到此橫幅。

## 2. 病人實付金額查詢（列表總額 + 歷史時間軸區間加總）

### 需求（已確認：兩者都要）

#### 2a. 病人管理列表（`PatientManagement`）
- 每位病人列新增一欄：**累計實付總額（全時段）**。
- 數值 = 該病人所有 `schedule_patients.actual_amount` 之和（不分時間）。

#### 2b. 病人歷史頁（`PatientHistory`）時間軸區間查詢
- 在病人歷史頁加上**日期區間選擇器**（RangePicker，依 `schedules.date`）。
- 選定區間後：
  - 歷史清單依區間篩選（只顯示該區間的看診紀錄）。
  - 顯示**該區間的實付總額**加總。
- 未選區間時：顯示全部歷史 + 全時段實付總額（與 2a 一致）。

### 驗收條件
- [ ] 病人列表每列顯示該病人全時段實付總額。
- [ ] 歷史頁可用日期區間篩選，並顯示該區間實付加總。
- [ ] 清空區間回到全部 + 全時段總額。
- [ ] 加總皆只計 `actual_amount`，`no_show` 不影響。

## 3. 排班建立時顯示病人「該年度已實付總額」

### 背景
排班建立/編輯 modal 內用 `SchedulePatientListEditor` + `PatientPicker` 選病人，每列可填 `prepaidAmount`。

### 需求
- 排班人員為某病人排班時，能看到**該病人在「排班所在年份」已實付的總額**。
  - 「年份」依正在編輯的排班 `date` 欄位的年份判定。
  - 總額 = 該病人於該年度（`schedules.date` 落在該年 1/1~12/31）所有 `actual_amount` 之和。

### 行為細節
- 在 `SchedulePatientListEditor` 每一列（已選定病人後）顯示，例如：`2026 年已實付：NT$ 8,000`。
- 病人未選定時不顯示。
- 排班日期改變導致年份改變時，數字需隨之更新。
- 此數字為**唯讀資訊**，不影響表單送出內容。

### 驗收條件
- [ ] 選定病人且排班日期有年份時，顯示該病人該年度實付總額。
- [ ] 改排班日期年份 → 數字更新。
- [ ] 未選病人不顯示；無資料顯示 NT$ 0。

### 決議（已確認）
- **不需任何特殊排除邏輯**：年度總額 = 查詢當下，DB 中該病人、該年度所有 `schedule_patients.actual_amount` 的加總，有多少算多少。
- 正在新增的這筆排班是尚未看診的全新資料，`actual_amount` 本就為 0，自然不影響；已存進 DB 的實付（不論翻譯員是否已按「離開」）都會被計入。

## 4. 病人 Excel 匯出附帶實付總額

### 需求
- 病人 Excel 匯出（`PatientService.BuildExcel`）每列**新增一欄：實付金額總額**（該病人全時段 `actual_amount` 之和）。
- **匯入維持不變**（匯入無此欄位，且原本欄位順序不可破壞匯入相容性）。

### 行為細節
- 新欄位加在現有欄位（姓名／電話／證件類型／證件號碼）**之後**，避免影響匯入時的欄位對應。
- 標題例：`實付金額總額`。
- 無紀錄者顯示 0。

### 驗收條件
- [ ] 匯出檔多一欄實付總額，數值正確。
- [ ] 既有匯入功能不受影響（欄位位置/解析不變）。
- [ ] 匯出範本（template）是否加此欄 → 範本**不加**（範本是給匯入用的）。

## 5. 排班頁面預設排序與「最新創建排班」按鈕

### 背景
目前後台排班列表 `schedule_repo.FindAll` 無分頁、固定 `date ASC, start_time ASC`。前端 filters：日期區間 / 翻譯員 / 地點。

### 需求
- **進入排班頁預設**：以**創建時間 `created_at` 由新到舊（DESC）**排序，**只取最近 100 筆**。
- 頁面上方新增一顆按鈕：**「最新創建排班」**。
  - 此按鈕代表「預設模式」（created_at DESC + 最近 100 筆）。
  - 處於預設模式時，按鈕**高亮**（active 樣式）。
- 互動規則：
  - 若套用任何**其他篩選**（日期區間 / 翻譯員 / 地點）→ 離開預設模式，按鈕**取消高亮**，列表改依該篩選查詢。
  - 若**清除所有其他篩選** → 回到預設模式（created_at DESC + 最近 100 筆）且按鈕**高亮**。
  - 若**按下「最新創建排班」按鈕** → 清掉其他篩選、回到預設模式（created_at DESC + 最近 100 筆）且按鈕**高亮**。

### 行為細節
- 「最近 100 筆」僅在預設模式套用；套用其他篩選時的筆數上限見下方待確認。
- 排序欄位 `created_at` 需從後端提供（目前 `FindAll` 排序為 `date ASC`）。

### 驗收條件
- [ ] 首次進入排班頁：created_at DESC、最多 100 筆、按鈕高亮。
- [ ] 套用任一其他篩選：按鈕取消高亮，結果依篩選。
- [ ] 清除所有其他篩選：回到預設且按鈕高亮。
- [ ] 按下按鈕：清除其他篩選、回到預設且按鈕高亮。

### 決議（已確認）
- 套用其他篩選時**維持現況**：`date ASC` 且**不限制筆數**。**只有預設模式**為 created_at DESC + 最近 100 筆。
- 「最近 100 筆」由**後端 limit**執行（避免回傳過多資料）。
